여러 타이머가 필요했지만 실제 동작은 mybatis 의 Mapper와 프로시져의 기능에 기대는 상황에서, 스프링 프레임워크의 동작을 모사하여 Dao 의 메소드를 만들고 custom annotation 을 적용하면 알아서 쓰레드들을 자동 생성하고 관리하는 구조를 만듦.

어노테이션 1: Dao의 메소드에 적용할 커스텀 어노테이션 정의.
어노테이션 2: Dao 인터페이스에 커스텀 어노테이션을 적용하여 리플렉션에 활용할 준비.
리플렉션 1: Dao의 클래스를 직접 참조해 메소드를 탐색하고, 그중 미리 정해놓은 커스텀 어노테이션이 적용된 메소드만을 이용해 쓰레드 생성 대상인 cocoRunner 의 생성자에 전달. 이 때 람다를 활용해 cocoRunner에서 직접 받은 메소드 알아서 활용하게끔 전달.
리플렉션 2: functional interface 중 하나인 Supplier 를 전달받는 생성자. 이후 전달받은 Supplier를 활용해 TimerDto를 생성.
